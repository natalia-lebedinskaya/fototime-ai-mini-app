# Test Strategy

## Цель тестирования

Проверить корректность работы Telegram Mini App для генерации AI-фото, включая получение конфигурации мероприятия, отображение участников и стилей, загрузку фото, mock-генерацию, негативные сценарии и обработку ошибок.

## Объект тестирования

- клиентская часть Telegram Mini App;
- backend-приложение;
- получение конфигурации мероприятия;
- отображение доступных участников;
- отображение доступных стилей;
- фильтрация стилей по выбранному участнику;
- форма загрузки изображения;
- API mock-генерации;
- обработка ошибок;
- поведение в Telegram WebApp окружении.

## Основная бизнес-логика

Стили и участники не должны быть захардкожены на frontend.

Приложение должно получать конфигурацию мероприятия с backend/API и отображать:

- только активных участников мероприятия;
- только стили, добавленные в мероприятие;
- только стили, подходящие выбранному участнику.

## Виды тестирования

- Functional testing;
- UI testing;
- Negative testing;
- Smoke testing;
- Regression testing;
- Cross-device testing;
- API testing;
- Error handling testing;
- Basic security checks.

## Критерии входа

Тестирование можно начинать, если:

- приложение запускается локально;
- сервер отвечает на endpoint `/api/health`;
- клиентская часть открывается в браузере;
- доступен endpoint получения конфигурации мероприятия;
- доступен endpoint mock-генерации;
- реализована базовая валидация формы;
- подготовлены тестовые изображения.

## Критерии выхода

Тестирование считается завершённым, если:

- smoke checklist пройден;
- основной пользовательский сценарий работает;
- критичные дефекты отсутствуют;
- негативные сценарии обработаны;
- API возвращает ожидаемые коды ошибок;
- документация актуальна.

## Scope

В рамках текущей версии проверяется:

- открытие приложения;
- получение mock-конфигурации мероприятия;
- отображение участников;
- отображение стилей;
- фильтрация стилей по выбранному участнику;
- выбор участника;
- выбор стиля;
- загрузка файла;
- клиентская валидация файла;
- серверная валидация файла;
- mock-flow генерации;
- обработка ошибок;
- базовая адаптивность.

## Out of Scope

На первом этапе не проверяется:

- реальная интеграция с NeuralSolutions API;
- реальное списание баланса;
- платёжная логика;
- авторизация пользователя;
- хранение истории генераций;
- отправка результата в Telegram-чат;
- production monitoring;
- полноценная админ-панель мероприятия.

## Проверки конфигурации мероприятия

| Проверка | Ожидаемый результат |
|---|---|
| Конфигурация мероприятия успешно получена | Отображаются участники и стили |
| Мероприятие не найдено | Пользователь видит понятное сообщение |
| Мероприятие неактивно | Генерация недоступна |
| Список участников пустой | Пользователь видит empty-state |
| Список стилей пустой | Пользователь видит empty-state |
| У выбранного участника нет доступных стилей | Отображается сообщение об отсутствии стилей |
| Backend вернул ошибку конфигурации | Пользователь видит retry-state |

## Проверки выбора участника

| Проверка | Ожидаемый результат |
|---|---|
| Пользователь выбирает участника | Участник визуально подсвечивается |
| Пользователь меняет участника | Выбор обновляется |
| При смене участника список стилей обновляется | Отображаются стили нового участника |
| При смене участника ранее выбранный неподходящий стиль сбрасывается | Невалидный стиль не остаётся выбранным |

## Проверки выбора стиля

| Проверка | Ожидаемый результат |
|---|---|
| Пользователь выбирает стиль | Стиль визуально подсвечивается |
| Стиль содержит previewUrl | Отображается превью |
| Стиль недоступен | Карточка disabled или скрыта |
| Стиль не подходит выбранному участнику | Стиль не отображается |
| Пользователь меняет стиль | В запрос уходит актуальный styleId |

## Проверки загрузки фото

| Проверка | Ожидаемый результат |
|---|---|
| JPG загружается | Файл принимается |
| JPEG загружается | Файл принимается |
| PNG загружается | Файл принимается |
| PDF загружается | Файл отклоняется |
| HEIC загружается | Файл отклоняется |
| Файл больше 10 MB | Файл отклоняется |
| Файл не выбран | Генерация недоступна |

## Проверки генерации

| Проверка | Ожидаемый результат |
|---|---|
| Все обязательные данные заполнены | Кнопка генерации активна |
| Не выбран участник | Кнопка генерации disabled |
| Не выбран стиль | Кнопка генерации disabled |
| Не загружено фото | Кнопка генерации disabled |
| Пользователь нажимает кнопку генерации | Запрос отправляется на backend |
| Пользователь быстро кликает несколько раз | Отправляется только один запрос |
| Backend вернул success | Отображается success-state |
| Backend вернул ошибку | Отображается понятное сообщение |

## Риски

| Риск | Потенциальное влияние | Как снизить |
|---|---|---|
| AI-сервис недоступен | Пользователь не сможет получить результат | Добавить обработку ошибок и retry |
| Медленная генерация | Пользователь может закрыть приложение | Добавить loading-state и timeout-message |
| Некорректный файл | Ошибка генерации | Добавить клиентскую и серверную валидацию |
| Конфигурация мероприятия не загрузилась | Пользователь не сможет выбрать стиль | Добавить empty/error/retry-state |
| Стили не соответствуют участнику | Пользователь выберет неподходящий стиль | Фильтровать стили по participantType |
| Дублирующий клик | Несколько запросов генерации | Блокировать кнопку после первого клика |
| Секреты попали в репозиторий | Риск компрометации токенов | Использовать `.env` и `.gitignore` |
