# Contributing

Thank you for helping improve FOT AI.

1. Create a focused branch from the default branch.
2. Keep user-facing changes multilingual where applicable.
3. Never commit `.env` files, user media, database exports, access tokens, or production identifiers.
4. Add or update tests for changed behaviour.
5. Run `npm run verify` before opening a pull request.
6. Describe the user impact, test evidence, known limitations, and screenshots for visual changes.

Bug reports should include environment, preconditions, exact steps, expected result, actual result, reproducibility, severity rationale, and safe supporting evidence.
